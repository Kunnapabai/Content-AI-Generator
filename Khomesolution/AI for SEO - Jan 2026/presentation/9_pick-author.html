Decision Algorithm

Deterministic scoring system ensuring exactly one category per keyword.

1

Preprocess Keyword

Trim whitespace and normalize to lowercase for matching. Original keyword text preserved for CLI flags and slug generation.

2

Score Each Category

Apply weighted scoring based on signal matches detected in the keyword text.

Primary/Core term +3
Type/Material match +2
Context cue +1
Negation detected -3
3

Resolve Ties

If multiple categories have the highest score, apply precedence order: ระแนง > หลังคา > ประตูหน้าต่าง. Then longest exact-match token wins.

4

Semantic Fallback

If all scores are zero or negative, apply semantic fallback: slat/screen terms go to ระแนง, roof terms to หลังคา, otherwise ประตูหน้าต่าง.
