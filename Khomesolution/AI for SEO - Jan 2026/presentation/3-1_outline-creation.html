Koray SEO Framework

Three pillars of semantic content structure for search-optimized outlines.

Macro Context

Exactly ONE central topic per page. The H1 establishes the single macro context that all other headings must support without drifting.

Single H1 Central Entity Source Context Root Node

Contextual Bridge

A required H2 that transitions from informational intent to decision/solution intent. Connects main content to monetization topics.

Info to Action Decision Point DIY vs Pro Selection Guide

Antonym Context

At least ONE supplementary H2 presenting the opposite perspective: drawbacks, limitations, or precautions for balanced coverage.

Drawbacks Limitations Precautions E-E-A-T Signal

Heading Tags Only

<h1> - Exactly one macro context
<h2> - Main sections (5+ minimum)
<h3> - Supporting subtopics
<h4> - Deep hierarchy (if needed)
No prose, no JSON, no markdown.
Pure HTML heading tags only.

Validation Checklist

Single Macro Context

Exactly one H1 tag. All headings support the same central topic without introducing a second macro context.

Contextual Bridge Present

Exactly one H2 that transitions from informational to decision intent (e.g., DIY vs professional).

Antonym Context Included

At least one supplementary H2 presenting drawbacks, limitations, or precautions for E-E-A-T compliance.

Language Consistency

All headings match the SERP language (Thai/English) detected in the analysis YAML file.

No Duplicate Headings

No duplicate heading strings across levels. FAQ questions do not repeat earlier H2 text.

Answer-First Structure

First H2 is phrased as a direct question mirroring H1's core intent for immediate answer delivery.
